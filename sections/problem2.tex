\section{问题二：最佳防守动作匹配}

\subsection{问题分析}

根据赛题设定，众擎 PM01 的防守行为包含格挡、闪避、防御姿态、平衡恢复、倒地应对及组合防守等类型。问题二要求针对附录一中的 13 种攻击动作给出最优防守响应。其本质可抽象为“攻击动作—防守动作”的条件匹配问题：在给定来袭攻击与当前状态的条件下，从可执行防守集合中筛选综合效用最高的动作或动作组合。

与仅关注“是否挡住攻击”的二元判断不同，机器人竞技中的防守质量应至少从以下四个维度综合评估：
\begin{enumerate}
    \item \textbf{完全防御能力}：能否通过格挡或闪避使攻击完全失效；
    \item \textbf{减伤能力}：在无法完全防住时，能否降低受击强度或减轻状态恶化程度；
    \item \textbf{反击衔接能力}：防守成功后，能否在对手收招或后摇期间形成有效反击窗口；
    \item \textbf{执行代价}：防守动作本身的体力消耗、姿态扰动及恢复成本是否可控。
\end{enumerate}

因此，问题二需要构建带状态约束的多指标评估模型，而非仅依赖经验规则进行动作枚举。

\subsection{动作库与状态空间定义}

为实现攻击动作与防守动作的自动匹配，本文基于动作库构建统一的攻防参数表。动作库中包含 13 种攻击动作与若干基础防守动作，每个动作均关联其可执行状态、状态转移倾向、基础命中/格挡概率、减伤映射、反击窗口概率以及体力消耗等参数；其中可直接由动作库读取的字段保留原值，其余难以直接获得的攻防概率参数则在后文通过经验先验与参数化随机模拟近似生成。

本文采用如下五状态离散表示机器人当前状态：
\[
\mathcal{S}=\{0,1,2,3,4\},
\]
其中
\[
\begin{aligned}
0 &= \text{Idle（空闲）},\\
1 &= \text{Guard（防御稳定）},\\
2 &= \text{MinorUnstable（轻度失稳）},\\
3 &= \text{MajorUnstable（重度失稳）},\\
4 &= \text{Down（倒地）}.
\end{aligned}
\]

这种状态划分能够较好反映机器人在受击、防守和恢复过程中的动态姿态变化。状态 \(0,1\) 表示机器人仍具有较好的攻防能力，状态 \(2,3\) 表示机器人已经出现不同程度的平衡破坏，状态 \(4\) 则表示机器人倒地，仅能执行地面防御或起身恢复类动作。

设攻击动作集合为
\[
\mathcal{A}_{\text{atk}}=\{a_1,a_2,\dots,a_{13}\},
\]
防守动作集合为
\[
\mathcal{A}_{\text{def}}=\{d_1,d_2,\dots,d_m\}.
\]
对于任一时刻的防守方状态 \(s\in\mathcal{S}\)，记在该状态下合法可执行的防守动作子集为
\[
\mathcal{A}_{\text{def}}(s)\subseteq \mathcal{A}_{\text{def}}.
\]

\subsection{防守评估模型}

对于来袭攻击 \(a\in\mathcal{A}_{\text{atk}}\)、候选防守动作 \(d\in\mathcal{A}_{\text{def}}(s)\)，定义以下四类核心指标：

\begin{itemize}
    \item \(p_{\text{block}}(a,d)\)：\textbf{完全防御概率}，表示防守动作 \(d\) 成功使攻击 \(a\) 失效的概率，包括有效格挡和有效闪避两种情形；
    \item \(p_{\text{mitigate}}(a,d)\)：\textbf{减伤概率}，表示防守动作虽然未能完全阻止攻击，但能够使防守方受击后的状态恶化程度下降一级的概率；
    \item \(p_{\text{counter}}(a,d)\)：\textbf{反击窗口概率}，表示在完成防守后，防守方已恢复为可继续行动状态，而攻击方仍处于攻击后摇、姿态暴露或收招恢复阶段的概率；
    \item \(c_{\text{self}}(d)\)：\textbf{防守执行代价}，表示执行防守动作 \(d\) 带来的体力消耗、姿态扰动及恢复成本。
\end{itemize}

综合以上因素，构建防守综合效用函数
\[
U_{\text{def}}(d\mid a,s)
=
w_1 p_{\text{block}}(a,d)
+
w_2 p_{\text{mitigate}}(a,d)
+
w_3 p_{\text{counter}}(a,d)
-
w_4 c_{\text{self}}(d),
\]
其中 \(w_1,w_2,w_3,w_4>0\) 为权重系数，分别表示完全防御、减伤、反击机会和防守代价在综合评价中的相对重要性。

在机器人竞技场景中，“防住攻击”通常优先级最高，因此取
\[
w_1>w_2,\qquad w_1\geq w_3,\qquad w_4>0.
\]
若比赛策略偏保守，则应适当提高 \(w_1,w_2\)；若策略偏主动压制，则可适当提高 \(w_3\)。

\subsection{指标参数化实现}

本文将动作库中的已有字段映射到上述效用函数参数中，实现从动作数据与近似参数库到防守评分的定量转换：

\begin{enumerate}
    \item \textbf{完全防御概率 \(p_{\text{block}}(a,d)\)}  
    由动作库中的动作类别、轨迹特征与防守机理共同映射为专项拦截概率。例如，“单手拍挡”对刺拳、后手直拳和前蹬腿分别具有较高的格挡能力；“下压格挡”对低扫腿具有更强的专项针对性；“转身闪避”则更适合应对回旋踢、侧踢、转身后蹬等大范围横向攻击。

    \item \textbf{减伤概率 \(p_{\text{mitigate}}(a,d)\)}  
    由减伤映射机制给出。若攻击未被完全挡住，但防守动作能将受击后的目标状态从“倒地”降为“重度失稳”、从“重度失稳”降为“轻度失稳”、或从“轻度失稳”降为“防御稳定”，则视为一次有效减伤。

    \item \textbf{反击窗口概率 \(p_{\text{counter}}(a,d)\)}  
    由防守动作自身的恢复属性、攻击动作的后摇特征以及经验先验共同近似确定。一般而言，位移型或闪避型防守动作虽然执行成本略高，但往往更容易在对手攻击结束后抢占节奏，形成反击机会。

    \item \textbf{执行代价 \(c_{\text{self}}(d)\)}  
    由防守动作的体力消耗给出，同时结合其自状态转移结果，用于衡量该动作是否容易导致自身失稳或后续行动能力下降。
\end{enumerate}

因此，问题二中的最佳防守匹配结果并非由完整高保真仿真实验直接输出，而是由动作库信息、经验先验与随机化参数库共同支撑的综合排序结果。

\subsection{参数生成方式说明}

需要说明的是，本文用于生成 \texttt{defense\_matching\_table.json} 的参数与结果，并非在完整高保真攻防仿真实验全部完成后直接统计得到。由于时间有限，全量动作组合、状态转移与多轮攻防链条的完整仿真尚未全部完成，因此问题二采用“经验先验 + 参数化随机模拟”的方式构造攻防参数库。

具体而言，可直接由动作库读取的动作合法性、基础消耗、恢复属性和部分状态转移信息保留为原始输入；对于现阶段难以精确获得的 \(p_{\text{block}}\)、\(p_{\text{mitigate}}\)、\(p_{\text{counter}}\) 等概率参数，则依据攻击轨迹、防守机理、动作恢复关系和竞技经验设置先验区间，再在概率取值范围、状态逻辑一致性和攻防相容性约束下进行带约束随机采样，并通过多次参数化随机模拟形成用于排序的近似参数库。

综上，本节结果应理解为在现有信息条件下得到的一组近似最优防守建议，其主要作用是为后续规律提炼与决策建模提供结构化输入。

\subsection{反击窗口的判定准则}

机器人竞技中的优秀防守不仅要求“防得住”，还要求“防完能接上”。因此，本文将反击窗口定义为：当防守动作结束时，若防守方处于可继续行动的状态集合
\[
\mathcal{S}_{\text{ready}}=\{\text{Idle},\text{Guard}\},
\]
而攻击方仍处于攻击后摇、姿态暴露或恢复中的脆弱状态集合
\[
\mathcal{S}_{\text{vul}}=\{\text{MinorUnstable},\text{MajorUnstable},\text{Down}\},
\]
则认为本次防守产生了有效反击窗口。

记防守完成后双方状态分别为 \(s'_{\text{self}}\) 和 \(s'_{\text{opp}}\)，则单次判定可表示为
\[
I^{\text{counter}}
=
\mathbf{1}\big(s'_{\text{self}}\in\mathcal{S}_{\text{ready}},
\ s'_{\text{opp}}\in\mathcal{S}_{\text{vul}}\big).
\]
在参数化随机模拟或参数库统计映射中，该指标取平均即可得到 \(p_{\text{counter}}(a,d)\)。

\subsection{防守动作合法性约束}

并非所有防守动作都可以在任意状态下执行。为保证模型结果具有物理可执行性，本文在计算防守效用前，先依据动作库中的合法状态集合 \(\texttt{legal\_from}\) 对候选动作进行筛选。

具体而言：
\begin{itemize}
    \item 常规格挡、闪避和防御姿态动作可在站立状态 \(0,1,2,3\) 下执行；
    \item “重心补偿”“步点调整”等恢复类动作主要适用于失稳状态 \(2,3\)；
    \item “受控倒地”只在重度失稳状态 \(3\) 下具备意义；
    \item “快速起身”“倒地防御”仅能在倒地状态 \(4\) 下执行。
\end{itemize}

因此，给定当前状态 \(s\)，实际参与排序的候选动作集合应写为
\[
\mathcal{A}_{\text{def}}(s)=\{d\in\mathcal{A}_{\text{def}}:\ d \text{ 在状态 } s \text{ 下合法}\}.
\]
该约束保证了模型输出不会出现“倒地时仍推荐十字格挡”之类不符合竞技物理现实的结果。

\subsection{最佳防御匹配算法}

基于上述建模思路，本文对每个攻击动作进行状态相关的防守匹配。算法的主要过程为：先根据当前状态筛选合法防守动作，再从参数库中读取或近似估计格挡概率、减伤映射、反击窗口和代价参数，计算综合效用并排序，最终输出前 \(K\) 个防守动作作为推荐集合。

\begin{algorithm}[H]
\caption{最佳防御匹配算法}
\label{alg:defense_match}
\begin{algorithmic}[1]
\Require 攻击集 \(\mathcal{A}_{\text{atk}}\)，防御集 \(\mathcal{A}_{\text{def}}\)，防守方当前状态 \(s\)，参数库
\Ensure 匹配表 \(M(a,s)\)

\For{每个攻击动作 \(a\in\mathcal{A}_{\text{atk}}\)}
    \State 根据当前状态 \(s\) 生成合法防守集合 \(\mathcal{A}_{\text{def}}(s)\)
    \For{每个防守动作 \(d\in\mathcal{A}_{\text{def}}(s)\)}
        \State 从参数库读取或近似估计 \(p_{\text{block}}(a,d)\)
        \State 根据减伤映射计算或近似估计 \(p_{\text{mitigate}}(a,d)\)
        \State 根据动作恢复与对手后摇估计 \(p_{\text{counter}}(a,d)\)
        \State 读取防守代价 \(c_{\text{self}}(d)\)
        \State 计算综合效用
        \[
        U_{\text{def}}(d\mid a,s)
        =
        w_1 p_{\text{block}}(a,d)
        +w_2 p_{\text{mitigate}}(a,d)
        +w_3 p_{\text{counter}}(a,d)
        -w_4 c_{\text{self}}(d)
        \]
    \EndFor
    \State 按 \(U_{\text{def}}\) 从大到小排序
    \State 取前 \(K\) 个动作作为推荐集合 \(M(a,s)\)
\EndFor
\State \Return \(M\)
\end{algorithmic}
\end{algorithm}

\subsection{最佳防御匹配结果}

利用上述方法，本文基于动作库已有信息、经验先验与带约束随机模拟所构造的参数库，分别计算了 13 种攻击动作在不同防守方状态下的近似最佳防守动作排序。近似结果表明，当防守方处于站立状态（空闲、稳定防御、轻度失稳、重度失稳）时，同一攻击动作对应的前三名防守动作排序基本保持一致；而当防守方处于倒地状态时，各攻击动作的最优防守统一转为“倒地防御”，次优动作统一为“快速起身”。这说明当前模型已经能够明显区分“站立防守”与“地面防守”两类对抗情形。

\subsection{本问题结论}

针对赛题中的 13 种攻击动作与动作库中的防守动作，本文建立了一个带状态约束的最佳防守匹配模型。模型以完全防御概率、减伤概率、反击窗口概率和防守代价为核心指标，综合比较各防守动作在当前状态下的适应性与收益，最终得到每种攻击动作的近似最优防守排序。

模型结果表明：直线型攻击宜采用快速格挡，弧线型攻击更适合专项闪避与肘挡，大范围腿法攻击宜优先规避，低段攻击则应采用专项低位防守；一旦机器人倒地，则应立即切换为地面防御与快速起身策略。上述规律不仅符合动作库给定的参数特征，也与机器人竞技中的实际攻防逻辑基本一致。因此，本文建立的攻击—防守近似匹配表可以作为问题三中单场攻防决策优化的先验输入。

\subsection{模型说明与进一步改进方向}

需要指出的是，在当前实现中，状态变量对排序结果的影响主要体现在动作合法性筛选层面。由结果可见，在状态 \(0\sim 3\) 下，同一攻击动作的前三名防守动作基本一致，仅在倒地状态 \(4\) 时出现显著变化。这表明模型已能区分“站立防守”与“倒地防守”两类情形，但对“空闲、稳定防御、轻度失稳、重度失稳”之间的细粒度差异刻画仍有提升空间。

后续可通过大规模完整仿真进一步校正 \(p_{\text{block}}\)、\(p_{\text{mitigate}}\)、\(p_{\text{counter}}\) 等概率参数，并将动作库中的自状态转移概率和攻击命中后的对手状态分布显式纳入评分函数，使不同失稳程度下的推荐动作更加差异化；同时还可以引入攻击距离、方位角、擂台边界位置等连续变量，将问题二从离散动作匹配进一步拓展为更贴近实战的状态相关最优防守决策模型。
